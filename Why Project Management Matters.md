# Why Project Management Matters

The single most important thing to internalize about working with Claude Code.

## The claim

> "Working with agentic AI is about 70 or 80% project management."

Claude Code does the work. *You* organize the work. If you don't organize the work, the powerful tool runs off and chews things up.

This isn't a complaint about Claude Code. It's a description of what Claude Code is: a very capable, very fast, very willing collaborator with no project memory of its own and no judgment about whether your direction is sound. Aim it well and you get superpowers. Aim it badly and you get a busy mess.

## The table-saw analogy

A handsaw is slow. A table saw is fast.

A table saw is also a motor with a giant blade attached. Without a table, without guards, without a fence, without a clear plan for which board you're cutting and where — it's a hazard wandering around your garage.

Claude Code is the table saw. Project management is the table, the fence, the guards, the marked board.

The point of the analogy isn't to scare you. It's to be honest about why the rest of this wiki exists. The tools below — projects-as-folders, the iteration loop, session logs, memory hygiene, commit cadence — are not bureaucracy. They are the table.

## What "on rails" means

When you're doing this well, you'll feel it: Claude Code asks the right next question, picks up where you left off, doesn't rebuild things you already finished, doesn't drift into adjacent territory unprompted. It's executing against a plan you reviewed.

When you're doing this badly, you'll also feel it: Claude builds something, asks "well, that wasn't quite right, what should we do?" and you say "I don't know, you figure it out." Then it unbuilds and rebuilds. Then it does it again. Then it's three days later and the project is a tangle.

The difference is not the model's intelligence. The difference is whether there's a written project plan, with you and Claude Code both reading from it.

## Project management as the limiting reagent

In the older worldview — "AI does the hard part, I just specify what I want" — the bottleneck is the AI. Make the AI smarter and outputs improve.

In Pete's worldview — "AI does the work, I do the project management" — the bottleneck is the human's project management practice. Make your project management better and outputs improve.

This is good news: it means you have a lot of leverage. It also means the ceiling on what you can accomplish is roughly the ceiling on how well you can scope, plan, review, and iterate.

If you have prior experience managing projects with humans — interns, contractors, teammates — most of it transfers. The same instincts apply: don't hand off without a brief, don't walk away without checking in, don't accept the first draft as the final draft.

## What this wiki is about

The rest of these pages are concrete, learnable habits that put Claude Code on rails. None are exotic. Most are what you'd do anyway with a competent intern; the difference is doing them deliberately, every time, because Claude Code can move fast enough that skipping them costs you a day instead of an hour.
